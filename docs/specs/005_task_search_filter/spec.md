# タスク検索・フィルタリング

## Goal

大量のタスクから目的のタスクを素早く見つけられるようにする。

## Non-goals

- 全文検索エンジンの導入（Elasticsearchなど）
- 保存済み検索条件（スマートリスト）
- 高度な検索クエリ構文
- 検索結果のハイライト表示
- タグ/カテゴリ検索（タグ機能実装後に追加）

## User Scenarios & Testing

- Given タスク一覧画面を開いている, When 検索ボックスにキーワードを入力する, Then タスク名と詳細が一致するタスクだけが表示される。
- Given 未完了タスクが含まれる, When 完了/未完了フィルタで「未完了」を選ぶ, Then 未完了タスクのみが表示される。
- Given 期限が今日・今週・期限切れ・期限なしのタスクがある, When 期限フィルタを切り替える, Then 選択した期限条件のタスクだけが表示される。
- Given 検索と複数フィルタを適用している, When いずれかの条件を変更する, Then すべての条件をANDで満たすタスクのみが表示される。
- Given 複数の条件を適用している, When フィルタをクリアする, Then 全タスクが表示される。

## Edge Cases

- 条件に一致するタスクが0件のときは、空の結果状態が表示される。
- 検索ボックスが空のときは、キーワード条件は未指定として扱われる。
- 期限が未設定のタスクは「期限なし」以外の期限フィルタでは表示されない。

## Requirements

### Functional Requirements

- FR-001 タスク一覧画面に検索ボックスを表示しなければならない。
- FR-002 検索ボックスの入力に応じて、タスク名と詳細で絞り込みを行わなければならない。
- FR-003 完了/未完了の状態でタスクをフィルタリングできなければならない。
- FR-004 期限（今日/今週/期限切れ/期限なし）でタスクをフィルタリングできなければならない。
- FR-005 複数のフィルタ条件をANDで組み合わせて適用できなければならない。
- FR-006 フィルタをクリアして全タスクを表示できなければならない。
- FR-007 入力変更やフィルタ変更により、結果が即時に更新されなければならない。
- FR-008 タグ/カテゴリなどの追加条件を将来的に取り込めるよう、検索・フィルタ条件は拡張可能でなければならない。

### Non-functional Requirements

- NFR-001 連続入力中でも画面操作が重くならない体験を提供しなければならない。
