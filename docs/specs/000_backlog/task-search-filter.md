# タスク検索・フィルタリング (Task Search & Filter)

## Goal
大量のタスクから目的のタスクを素早く見つけられるようにする。

## User story
- ユーザーはタスク名や詳細でキーワード検索できる。
- ユーザーは完了/未完了でタスクをフィルタリングできる。
- ユーザーは期限（今日/今週/期限切れ/期限なし）でタスクをフィルタリングできる。
- ユーザーはフィルタ条件を組み合わせて使用できる。

## Non-goals
- 全文検索エンジンの導入（Elasticsearchなど）
- 保存済み検索条件（スマートリスト）
- 高度な検索クエリ構文

## Acceptance criteria
- [ ] タスク一覧画面に検索ボックスがあること。
- [ ] 検索ボックスに入力すると、タスク名と詳細がリアルタイムで絞り込まれること。
- [ ] フィルタドロップダウンまたはボタンで条件を選択できること。
- [ ] 複数のフィルタ条件をAND結合で適用できること。
- [ ] フィルタをクリアして全タスクを表示できること。

## Notes
- 検索はフロントエンドでのフィルタリング（クライアントサイド）で初期実装。
- タスク数が多くなった場合はAPIでのサーバーサイド検索を検討。
- デバウンス処理を入れてパフォーマンスを確保。

## Open Questions
- 検索対象にタグ/カテゴリも含めるか？ -> タグ機能実装後に拡張。
- 検索結果のハイライト表示は必要か？ -> Nice to have、MVPでは不要。
