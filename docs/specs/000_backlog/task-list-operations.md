# タスク一覧操作の効率化 (Efficient Task List Operations)

## Goal

タスク数が増えても一覧性と操作速度を維持できるようにし、日々の優先順位変更と消化を素早く行える状態を作る。

## User story

- ユーザーはタスクのテキストをその場で編集し、編集のために別画面やモーダルへ移動したくない。
- ユーザーはドラッグ＆ドロップでタスクの表示順を直感的に変更したい。
- ユーザーは完了済みタスクを未完了タスクと分離して、今やるべき項目に集中したい。

## Non-goals

- ユーザー間で共有されるリアルタイム同時編集
- AIによる優先度自動算出
- ガントチャートやカレンダーのような別ビュー追加

## Acceptance criteria

- [ ] タスク名をクリックするとインライン編集モードに入り、その場で保存できること。
- [ ] インライン編集は保存操作とキャンセル操作の両方をサポートすること。
- [ ] ドラッグ＆ドロップでタスク順序を変更でき、再表示後も順序が維持されること。
- [ ] タスクを完了状態にすると、未完了タスクとの視覚的区別（例: グレーアウト）が即時に反映されること。
- [ ] 完了済みタスクは未完了タスクの集中を妨げない配置（下部移動または別タブ）で表示されること。

## Notes

- 一覧操作の改善は、可読性よりも「編集・並び替え・整理の連続操作が止まらないこと」を重視する。
- 完了済みタスクの表示方式は、運用しながらチームに合う形へ調整可能な設計が望ましい。

## Open Questions

- 完了済みタスクは「下部に移動」と「別タブ」のどちらをデフォルトにするか？
- ドラッグ＆ドロップ順序は手動優先度として扱うか、別の優先度機能と併用するか？
- インライン編集時のキーボード操作仕様（Enter/Escape/Tab）をどこまで標準化するか？
