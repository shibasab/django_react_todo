# 繰り返しタスク設定 (Recurring Task Management)

## Goal

定期的に発生するタスクを繰り返し設定できるようにし、手動での再作成負荷と対応漏れを減らす。

## User story

- ユーザーはタスクに日次・週次・月次などの繰り返しルールを設定したい。
- ユーザーは繰り返しルールを後から見直し、変更または停止したい。
- ユーザーは同じタスクを毎回手入力せずに継続運用したい。

## Non-goals

- カンバンボードやドラッグ＆ドロップによる可視化
- チーム向けの担当者割り当てや共同編集
- 繰り返しルールの高度な式設定（例: 複雑なカレンダールール）

## Acceptance criteria

- [ ] タスク作成・編集時に繰り返し設定を付与できること。
- [ ] 繰り返し設定は少なくとも日次・週次・月次の基本周期を扱えること。
- [ ] 繰り返し設定されたタスクを継続的に管理できること。
- [ ] 既存タスクの繰り返し設定を更新・解除できること。
- [ ] 繰り返し設定の有無をユーザーが識別できること。

## Notes

- MVPではシンプルな周期設定から開始し、実運用に応じて拡張する。

## Open Questions

- 次回タスクの生成タイミングは「完了時」か「事前生成」か？
- 期限を過ぎた未完了タスクがある場合、次回分の扱いをどうするか？
- タイムゾーンや日付切り替え境界の扱いをどこまで定義するか？
