# タスク進捗カンバンボード (Task Progress Kanban Board)

## Goal

タスク進捗を「着手前・進行中・完了」で可視化し、カンバンボード上で直感的に更新できるようにする。

## User story

- ユーザーはタスクを状態別の列で一覧し、今どこまで進んでいるかを把握したい。
- ユーザーはタスクをドラッグして列移動し、進捗状態を素早く更新したい。
- ユーザーは一覧表示とカンバン表示で同じ進捗状態を一貫して確認したい。

## Non-goals

- 工数見積もり、バーンダウン、ガントチャートなどのプロジェクト管理機能
- AIによる進捗自動判定
- 複数ユーザーでのリアルタイム共同編集

## Acceptance criteria

- [ ] タスクに「着手前・進行中・完了」の進捗状態を保持できること。
- [ ] 進捗状態に対応したカンバン列でタスクを表示できること。
- [ ] カンバン上で列移動すると進捗状態が更新されること。
- [ ] 一覧表示とカンバン表示で進捗状態が一致すること。
- [ ] 進捗状態の変更結果が再表示後も維持されること。

## Notes

- 既存の「完了ステータス」機能との整合性を取りつつ段階的に拡張する。

## Open Questions

- カンバン列は固定3列で開始するか、将来的な列カスタマイズを前提にするか？
- 完了列のタスク表示量が増えた場合の扱い（折りたたみ、別表示など）をどうするか？
- 進捗状態の名称は運用に応じて変更可能にするか？
