# タスク完了ステータス (Task Completion Status)

## Goal
タスクの完了/未完了を管理できるようにし、進捗状況を可視化する。

## User story
- ユーザーはタスクを完了としてマークできる。
- ユーザーは完了したタスクを未完了に戻せる。
- ユーザーはタスク一覧で完了/未完了の状態を視覚的に確認できる。
- ユーザーは完了タスクと未完了タスクをフィルタリングして表示できる。

## Non-goals
- 進捗率（パーセンテージ）の管理
- サブタスクの完了状態管理
- 完了日時の自動記録（将来的には検討）

## Acceptance criteria
- [ ] タスクモデルに `is_completed` (完了フラグ) カラムが追加されていること。
- [ ] API経由でタスクの完了状態を更新できること。
- [ ] フロントエンドでワンクリックで完了/未完了を切り替えられること。
- [ ] タスク一覧で完了タスクが視覚的に区別されること（取り消し線、グレーアウトなど）。

## Notes
- 完了状態の切り替えは専用のAPIエンドポイント（PATCH）を設けるか、既存のPUTを利用するか要検討。
- UIはチェックボックスまたはトグルボタンを想定。

## Open Questions
- 完了タスクをデフォルトで非表示にするか、表示するか？ -> 初期実装では表示し、フィルタで切り替え可能とする。
- 完了日時を記録するか？ -> MVPでは不要。将来的には追加検討。
