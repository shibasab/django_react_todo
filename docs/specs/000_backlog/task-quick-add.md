# タスク登録の摩擦を減らすクイック追加 (Task Quick Add)

## Goal

思いついたタスクを最短操作で登録できるようにし、日常的に使いたくなる入力体験を作る。

## User story

- ユーザーはタスク一覧の上部にある1行入力バーへ、思いついた内容をすぐ入力して登録したい。
- ユーザーは「来週の金曜日にリリース #work」のような自然文を入力し、期日やタグを自動反映したい。
- ユーザーは詳細入力のためにフォームを何項目も埋めなくても、まずタスクを保存したい。

## Non-goals

- すべての自然言語表現を高精度に解釈すること
- 音声入力やOCRなど、テキスト入力以外の新規入力手段
- 定期タスクや高度なスケジューリングルールの同時実装

## Acceptance criteria

- [ ] タスク一覧の上部に、クイック追加用の1行入力UIが表示されること。
- [ ] 1行入力で送信すると、タスクが即時に作成されること。
- [ ] 自然文から、少なくとも基本的な期限表現（例: 今日、明日、来週金曜）を抽出できること。
- [ ] `#tag` 形式の入力からタグ情報を抽出できること。
- [ ] 解釈できない語句が含まれても登録が失敗せず、最低限タスク名として保存されること。

## Notes

- クイック追加は「入力開始までの操作数を最小化する」ことを最優先にする。
- 既存の詳細入力フローが必要な場合は、クイック追加後の編集で補完できる形を想定する。

## Open Questions

- 既存の複数フィールドフォームは廃止するか、併存させるか？
- 自然文パースの初期対応範囲をどこまでにするか？
- タグ未登録時に自動作成を許可するか？
