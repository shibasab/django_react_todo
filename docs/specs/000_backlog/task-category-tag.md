# タスクカテゴリ・タグ (Task Category / Tag)

## Goal
タスクを分類・整理し、関連するタスクをグループ化して管理できるようにする。

## User story
- ユーザーはタスクに1つ以上のタグを付けられる。
- ユーザーは自分専用のタグを作成・編集・削除できる。
- ユーザーはタグでタスクをフィルタリングできる。
- ユーザーはタスク一覧でタグを視覚的に確認できる。

## Non-goals
- 階層構造を持つカテゴリ
- 共有タグ（他ユーザーとの共有）
- タグの色のカスタマイズ（将来的には検討）

## Acceptance criteria
- [ ] タグを管理するための新規モデル/テーブルが作成されていること。
- [ ] タスクとタグの多対多のリレーションが実装されていること。
- [ ] タグのCRUD APIが存在すること。
- [ ] タスク作成・編集時にタグを選択/入力できること。
- [ ] タスク一覧でタグがバッジとして表示されること。
- [ ] タグをクリックするとそのタグのタスクでフィルタリングされること。

## Notes
- タグはシンプルな文字列ラベルとして実装。
- 既存タグの候補表示（オートコンプリート）があるとUX向上。
- タグの上限は1タスクあたり5個程度を想定。

## Open Questions
- タグの色分けは必要か？ -> MVP後に検討。自動割り当てまたはユーザー指定。
- プリセットタグ（「仕事」「プライベート」など）を用意するか？ -> ユーザーが自由に作成する形式でスタート。
- カテゴリとタグを別概念として持つか？ -> MVPではタグのみ。カテゴリは階層が必要になった場合に検討。
