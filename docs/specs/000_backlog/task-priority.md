# タスク優先度設定 (Task Priority)

## Goal
タスクに優先度を設定し、重要なタスクを識別・管理しやすくする。

## User story
- ユーザーはタスク作成時または編集時に優先度を設定できる。
- ユーザーはタスク一覧で優先度が高いタスクを視覚的に識別できる。
- ユーザーはタスク一覧を優先度でソートできる。

## Non-goals
- 優先度の自動計算（期限との組み合わせなど）
- カスタム優先度レベルの定義
- 優先度に基づくリマインダー

## Acceptance criteria
- [ ] タスクモデルに `priority` カラムが追加されていること。
- [ ] 優先度は3段階（高/中/低）または4段階（緊急/高/中/低）で設定可能であること。
- [ ] フロントエンドで優先度を選択できるUIがあること。
- [ ] タスク一覧で優先度がアイコンや色で視覚的に表示されること。
- [ ] 優先度でソートできること。

## Notes
- 優先度のデフォルト値は「中」または「低」とする。
- UIはドロップダウンまたはアイコンボタンを想定。
- 色分け例：高=赤、中=黄、低=緑（またはグレー）

## Open Questions
- 優先度レベルは何段階がよいか？ -> シンプルに3段階（高/中/低）でMVP。
- 優先度なし（未設定）を許可するか？ -> デフォルト値を設定し、未設定は許可しない方向。
