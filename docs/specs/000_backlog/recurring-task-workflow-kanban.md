# 繰り返しタスクと進捗カンバン管理 (Recurring Task Workflow & Kanban)

## Goal

繰り返し発生するタスクの抜け漏れを減らし、タスクの進捗を「着手前・進行中・完了」で一目で把握しながら直感的に更新できる状態を作る。

## User story

- ユーザーは定期的に発生するタスクを繰り返し設定し、毎回手動で再作成したくない。
- ユーザーは各タスクの進捗状態を「着手前・進行中・完了」で管理したい。
- ユーザーはカンバンボード形式でタスクを参照し、ドラッグ操作で列間移動して進捗を更新したい。

## Non-goals

- 複数ユーザーでの共同編集や担当者アサイン
- 工数見積もり、バーンダウン、ガントチャートなどのプロジェクト管理機能
- 進捗の自動判定やAIによるタスク分解

## Acceptance criteria

- [ ] タスクに繰り返し設定を付与できること（少なくとも日次・週次・月次などの基本周期を選択可能）。
- [ ] 繰り返し設定されたタスクが、継続的に管理できる形で一覧に反映されること。
- [ ] タスクごとに「着手前・進行中・完了」の進捗状態を保持・更新できること。
- [ ] 進捗状態は一覧表示とカンバン表示の両方で一貫して確認できること。
- [ ] カンバン上でタスクを列移動すると、対応する進捗状態が更新されること。

## Notes

- 既存の完了概念との整合性を取りつつ、段階的に進捗管理へ拡張する方針が望ましい。
- 繰り返し設定とカンバン表示は利用価値が高いが、導入順は別途検討の余地がある。

## Open Questions

- 繰り返しタスクは「完了時に次回分を生成」か「事前に複数生成」か、どちらを採用するか？
- 期限超過した繰り返しタスクの扱い（繰越・スキップ・自動クローズ）をどうするか？
- カンバン列の定義は固定（3列）か、将来的にユーザーがカスタマイズ可能にするか？
