# サブタスク作成機能 (Subtask Creation)

## Goal

大きなタスクをサブタスクへ分解したときでも、親子関係を見失わずに進捗を把握できるようにし、作業の見通しを改善する。

## User story

- ユーザーは大きなタスクを実行可能な小さな作業に分解し、親タスクに紐づくサブタスクとして管理したい。
- ユーザーはサブタスクの完了状況から、親タスクがどの程度進んでいるかを割合で把握したい。
- ユーザーはサブタスク一覧を見て「この作業が何のための作業か」をすぐ理解したい。

## Non-goals

- サブタスクの多階層化（サブタスクのさらに子タスク）
- サブタスク依存関係（先行・後続）の管理
- AIによるサブタスク自動生成

## Acceptance criteria

- [ ] 任意のタスクに対してサブタスクを追加できること。
- [ ] サブタスクには親タスクIDまたは親タスクへの参照情報が保持されること。
- [ ] サブタスク一覧で親タスク名（または親タスクへの導線）が確認できること。
- [ ] 親タスク画面で紐づくサブタスク一覧が表示されること。
- [ ] サブタスクの完了・未完了が親タスクの進捗率（例: 3/5, 60%）に反映されること。
- [ ] サブタスクの状態変更時、親タスク進捗率が再計算されること。

## Notes

- 進捗率の基本計算は「完了サブタスク数 / 全サブタスク数」とする。
- 親タスクにサブタスクが存在しない場合の進捗表示は要検討（未表示・0%表示など）。
- 親子関係の視認性を高めるため、UI上の導線（パンくず、リンク、バッジ等）を検討する。

## Open Questions

- 親タスク自身を完了にする条件は、サブタスク完了率100%を必須にするか？
- 親タスクに「手動進捗」と「サブタスク自動進捗」を併用可能にするか？
- サブタスクの並び順（作成順、手動並び替え、期限順）をどの方式にするか？
