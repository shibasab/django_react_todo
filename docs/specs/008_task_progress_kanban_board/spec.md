# タスク進捗カンバンボード

## Goal

タスク進捗を「着手前・進行中・完了」の3状態で可視化し、一覧表示とカンバン表示のどちらでも一貫した状態管理を可能にする。

## Non-goals

- 工数見積もり、バーンダウン、ガントチャートなどのプロジェクト管理機能
- AIによる進捗自動判定
- 複数ユーザーのリアルタイム共同編集
- 任意数のカンバン列カスタマイズ

## User Scenarios & Testing

### Scenario 1: カンバンで進捗を俯瞰する

- Given 進捗状態が異なる複数のタスクが登録されている
- When ユーザーがカンバン表示を開く
- Then タスクは「着手前」「進行中」「完了」の対応列に表示される

### Scenario 2: 列移動で進捗を更新する

- Given ユーザーがカンバン表示を開いている
- When ユーザーがタスクを別列へ移動する
- Then 対応する進捗状態が更新される

### Scenario 3: 一覧表示との整合を確認する

- Given カンバン表示でタスクの進捗状態を更新した
- When ユーザーが一覧表示を開く
- Then 同じタスクに同じ進捗状態が表示される

### Scenario 4: 一覧表示から変更した内容が反映される

- Given 一覧表示でタスクの進捗状態を更新した
- When ユーザーがカンバン表示を開く
- Then 対応列に更新後の状態で表示される

### Scenario 5: 再表示後も状態を維持する

- Given ユーザーがタスクの進捗状態を変更した
- When 画面再表示またはアプリ再起動後に同じタスクを確認する
- Then 変更後の進捗状態が維持されている

## Edge Cases

- 完了タスクが多数ある場合でも、完了列に状態不整合なく表示されること
- 進捗未設定の既存タスクが存在する場合、初期状態が一意に決定されること
- 状態更新の途中で表示を切り替えた場合でも、二重更新や不整合が発生しないこと

## Requirements

### Functional Requirements

- FR-001: システムは各タスクに対して「着手前」「進行中」「完了」のいずれか1つの進捗状態を保持する（MUST）。
- FR-002: システムはカンバン表示において、進捗状態ごとに3列を提供する（MUST）。
- FR-003: システムはタスクの列移動操作に応じて、対応する進捗状態を更新する（MUST）。
- FR-004: システムは一覧表示とカンバン表示で、同一タスクの進捗状態を一貫して表示する（MUST）。
- FR-005: システムは進捗状態の変更結果を再表示後も保持する（MUST）。
- FR-006: システムは既存の完了ステータスとの意味不一致を発生させない（SHOULD）。

### Non-functional Requirements

- NFR-001: 進捗状態の更新操作は、通常操作においてユーザーに知覚可能な遅延を発生させない（SHOULD）。
- NFR-002: 表示切り替え後の状態整合性は、データ不整合を発生させないことを満たす（MUST）。

## Decisions

- D-001: 内部状態モデルは今回の時点で拡張可能な形にし、UIは固定3列のままとする。
- D-002: 完了列の表示最適化は初期リリースの必要要件には含めず、利用実績を見て次フェーズで検討する。
- D-003: 進捗状態ラベルは初期リリースでは固定ラベル（着手前/進行中/完了）とする。
